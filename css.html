CSS part I

Co to jest CSS?

CSS Cascading Style Sheets - Kaskadowe Arkusze Stylów:

  • Odpowiadają za prezentację treści strony, wygląd elementów HTML na ekranie,wydrukach itp.
  • Nie mają jednej wersji (rozwój modułów), potocznie mówi się, że aktualna wersja to 3.
  • Przeglądarki potrafią różnie interpretować CSS co prowadzi do tego, że pisanie stylów powoduje wiele frustracji u programistów (ale jest też niezłym wyzwaniem :)).
  • Całą specyfikację znajdziesz na stronie W3C.

Składnia CSS to reguły

selektor {
właściwość: wartość;
}

Pliki z rozszerzeniem .css -> ale nie jest to jedyna reguła
Podpinamy w sekcji head <link href=”style.css” rel=”stylesheet” type=”text/css”> Można stylować w sekcji head <style>

CSS part II

Jak można stylować?

  • W sekcji head <style>
  • Style inline <znacznik style=”color:yellow;”>treść</znacznik>
  • Przy użyciu !important
  • Po elementach: element
  • Po klasach: .class (. kropka to selektor klasy)
  • Po id: #id (# hasz to selektor id)

Klasy

html class=”name = css .name
html id=”name” =css #name

Najlepiej stylować po klasach, bez zagnieżdżeń.
Klasy nazywamy po angielsku zgodnie z przeznaczeniem, tworzymy klasy nawet kiedy mamy jeden element danego typu.
Nie używamy important i stylów inlineowych.

Klasa w HTML:
<znacznik class=”name”>treść</znacznik>

Klasa w CSS:
.name {
właściwość: wartość;
}

Nazewnictwo klas:

  • Używać poj. słów z j. angielskiego
  • Jeśli już muszą być dwa słowa użyć kebab-case
  • Nazywać rzeczy zgodnie z ich funkcją na stronie
  • Zaczynać od małej litery, nie zaczynać od cyfr
  • Starać się, żeby element miał jedną klasę - nie zwiększać specyficzności bez potrzeby

Łączenie selektorów

Na przykład wszystkie linki z nav
nav a { albo header a {
color: blue;
}
Style zaaplikują się tylko do konkretnego “a” z nav

a {
color: red;
}
Wszystkie inne “a” na stronie, które nie zawierają się w nav (header)
!important - nadpisuje style wszystkich “a”, lepiej nie używać :D

Specyficzność w CSS

Jak są przejmowane style? Które są ważniejsze? Które mają wyższą specyficzność?

Jak to liczyć?

Jeżeli specyficzność jest równa - pierwszeństwo ma selektor “bliżej elementu”, bliżej dołu strony.

Najlepiej dodawać klasy do wszystkich elementów i unikać zagnieżdżeń! Najłatwiej jest wtedy stylować :).
Style, które ustawiasz na konkretny element mają pierwszeństwo nad tymi, które się dziedziczy
